# Dispatch: /webpage pillars (Website Updates)

## Process

1. **Read the PMM brief.**

2. **Run `/webpage new "[Page Name]"`** with the brief's content as context. The `/webpage` command handles the multi-agent design process.

3. **Output path**: Set the `/webpage` project path to `content/launches/[launch-slug]/website/` instead of the default `webpage-builder/projects/` location.

4. **Update tracker.** Set status to `complete`, output path to `content/launches/[slug]/website/`.
